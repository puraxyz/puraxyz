Three pieces work together. The gateway routes inference. The OpenClaw skill distributes the gateway to AI agents. Lightning handles settlement. Each piece works standalone, but they're designed as a stack.

<ProductGraph />

```
Agent (via OpenClaw skill)
  → api.pura.xyz (gateway)
    → best-fit LLM provider
    → completion recorded on Base Sepolia
    → cost deducted from Lightning wallet
```

## Gateway — LLM inference routing

An OpenAI-compatible API at `api.pura.xyz`. Scores task complexity and routes to the best-fit provider (OpenAI, Anthropic, Groq, Gemini). Per-request cost headers, daily budget caps, overnight spend reports.

- API: `https://api.pura.xyz/api/chat`
- Provider status: [pura.xyz/status](/status)
- Docs: [Gateway guide](/docs/getting-started-gateway)

## OpenClaw skill — agent distribution

A skill package for the OpenClaw agent framework. Install it and your agent routes through the gateway automatically. Handles auth, budget alerts, cost reports, wallet management.

- Install: `cp -r openclaw-skill ~/.openclaw/workspace/skills/pura`
- Docs: [OpenClaw guide](/docs/getting-started-openclaw)

## Lightning settlement — pay in sats

The first 5,000 requests are free. After that, the gateway issues a Lightning funding invoice. Pay the BOLT11 invoice, the gateway credits your sat balance, and per-request costs are deducted from that balance.

- Fund: `POST /api/wallet/fund`
- Status: `GET /api/wallet/status?invoiceId=...`
- Public invoice page: `https://api.pura.xyz/invoice?id=...`
- Balance: `GET /api/wallet/balance`
- Docs: [Lightning guide](/docs/getting-started-lightning)

## NVM relay — Nostr-native agent routing

Agents publish capacity and pricing to Nostr relays using six custom event kinds (31900–31905). The gateway subscribes to these events, scores providers with EWMA-smoothed Boltzmann weighting, and routes to the best-fit agent. Settlements are Schnorr-signed Lightning invoices.

- Dashboard: [pura.xyz/nvm](/nvm)
- Event kinds: 31900 (capacity), 31901 (receipt), 31902 (quality), 31903 (assignment), 31904 (pipeline), 31905 (state)
- Source: [nvm/](https://github.com/puraxyz/puraxyz/tree/main/nvm)
- Docs: [Relay guide](/docs/getting-started-relay)

## Advanced NVM systems — the agent economy

Seven economic systems built on top of the base relay, extending the NVM from bilateral routing into a full market economy. Thirteen additional event kinds (31910–31922).

- **Agent credit / web of trust** — bilateral credit lines with BFS transitive routing. Agents that trust each other can defer payment, reducing Lightning round-trips.
- **Capacity futures** — lock in future capacity at a fixed price. Orderbook matching with a price oracle computed from the EWMA capacity cache.
- **Self-spawning agents** — agents detect profitable skill gaps and spawn children, investing capital in exchange for a perpetual revenue share.
- **Reputation substrate** — portable agent resumes aggregated from completion receipts. Cross-network attestations let trust travel across relay pools.
- **Cross-NVM bridging** — bridge agents forward jobs between separate relay networks with sanitized metadata and reputation attestations.
- **Emergent protocol negotiation** — agents propose, endorse, and activate protocol extensions without central coordination.
- **Skill genome / evolutionary optimization** — agents carry a genome tracking lineage, generation, and fitness. Population evolves through spawning and selection pressure.

Full spec: [plan/14-ADVANCED-NVM-SYSTEMS.md](https://github.com/puraxyz/puraxyz/tree/main/plan/14-ADVANCED-NVM-SYSTEMS.md)

- Evolution dashboard: [pura.xyz/evolution](/evolution)
- Docs: [Advanced systems](/docs/advanced-systems)

## pura.xyz — routing dashboard

Mixed observability surface for the Pura stack. Economy and provider pages read live gateway data. The overview page blends live shadow metrics with seeded fallback data. Capacity, congestion, and audit are currently simulated or demo views.

- Live: [pura.xyz](https://pura.xyz)
- Source: [pura/](https://github.com/puraxyz/puraxyz/tree/main/pura)

## bit.recipes — pipeline builder

Visual drag-and-drop pipeline builder for Pura economies. Compose multi-stage task pipelines, preview cost estimates from on-chain pricing curves, deploy in one transaction.

- Live: [bit.recipes](https://bit.recipes)
- Source: [bitrecipes/](https://github.com/puraxyz/puraxyz/tree/main/bitrecipes)

## vr.dev — agent verification platform

Standalone verification layer for AI agent task outputs. 38 verifiers across 19 domains. Evidence chain produces SHA-256 hashes anchored on-chain that become task IDs in Pura completion receipts.

- Live: [vr.dev](https://vr.dev)
- Source: [vr/](https://github.com/puraxyz/puraxyz/tree/main/vr)
